# Architecture

## Overview
PYE is a Next.js application using TypeScript, Tailwind CSS, and a custom design system.

## Directory Structure

```
src/
├── app/              # Next.js App Router pages and layouts
├── components/       # Shared, reusable UI components
├── features/         # Domain-specific feature modules (service + hook + test)
├── design-system/    # Design tokens and UI rules
├── lib/              # Third-party library wrappers and clients
├── utils/            # Pure utility/helper functions
└── tests/            # Integration and end-to-end tests
```

## Layers

| Layer           | Responsibility                                      |
|-----------------|-----------------------------------------------------|
| `app/`          | Routing, layout, page composition                   |
| `components/`   | Stateless or lightly stateful presentational UI     |
| `features/`     | Business logic, API calls, local state (per domain) |
| `lib/`          | External integrations (API clients, SDKs)           |
| `utils/`        | Stateless pure helper functions                     |
| `design-system/`| Tokens, spacing, colors, typography rules           |

## Data Flow
Pages (`app/`) → Feature hooks (`features/*/auth.hook.ts`) → Services (`features/*/auth.service.ts`) → API / External

## Tech Stack
- **Framework**: Next.js (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS + design tokens
- **Testing**: Jest + React Testing Library
- **CI**: GitHub Actions
