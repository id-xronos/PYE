# AI Rules

## General Guidelines
- Follow the architecture defined in `.architecture.md`
- Use TypeScript for all source files
- Prefer functional components and hooks for React code
- Write tests for all business logic

## Code Style
- Use meaningful variable and function names
- Keep functions small and focused (single responsibility)
- Avoid magic numbers and strings; use constants or tokens
- Use the design system tokens from `src/design-system/tokens.ts`

## Component Rules
- All components live in `src/components/`
- Feature-specific components live within their feature folder
- Components must be typed with explicit props interfaces

## Testing
- Unit tests live alongside the code they test (`.test.ts` / `.test.tsx`)
- Integration tests live in `src/tests/`
- Aim for >80% coverage on business logic
